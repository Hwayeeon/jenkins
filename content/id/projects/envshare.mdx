EnvShare adalah alat sederhana untuk berbagi variabel environment secara aman. Menggunakan
**AES-GCM** untuk mengenkripsi data Anda sebelum mengirimnya ke server. Kunci enkripsi
tidak pernah meninggalkan browser Anda.

## Fitur

- **Link yang Dapat Dibagikan:** Bagikan variabel environment Anda secara aman dengan mengirimkan
  link
- **Enkripsi End-to-End:** Enkripsi AES-GCM digunakan untuk mengenkripsi data Anda
  sebelum mengirimnya ke server
- **Batasi jumlah pembacaan:** Batasi berapa kali sebuah link dapat dibaca
- **Auto Expire:** Secara otomatis kedaluwarsa link dan menghapus data setelah waktu tertentu

### API

#### Simpan secret

**JANGAN PERNAH UPLOAD SECRET YANG TIDAK TERENKRIPSI.**

Endpoint ini hanya dimaksudkan untuk menyimpan secret yang **sudah terenkripsi**. Secret yang
terenkripsi disimpan dalam bentuk plain text.

```sh-session
$ curl -XPOST -s https://envshare.dev/api/v1/secret -d "already-encrypted-secret"
```

Anda dapat menambahkan header opsional untuk mengkonfigurasi ttl dan jumlah pembacaan.

```sh-session
$ curl -XPOST -s https://envshare.dev/api/v1/secret -d "already-encrypted-secret" -H "envshare-ttl: 3600" -H "envshare-reads: 10"
```

- Menghilangkan header `envshare-ttl` akan menetapkan default 30 hari. Nonaktifkan
  ttl dengan menetapkannya ke 0. (`envshare-ttl: 0`)
- Menghilangkan header `envshare-reads` akan menonaktifkannya dan memungkinkan pembacaan
  untuk jumlah yang tidak terbatas.

Endpoint ini mengembalikan respons JSON dengan id secret:

```json
{
  "data": {
    "id": "HdPbXgpvUvNk43oxSdK97u",
    "ttl": 86400,
    "reads": 2,
    "expiresAt": "2023-01-19T20:47:28.383Z",
    "url": "http://envshare.dev/api/v1/secret/HdPbXgpvUvNk43oxSdK97u"
  }
}
```

#### Ambil secret

Anda membutuhkan id untuk mengambil secret. Id dikembalikan saat Anda menyimpan secret.

```sh-session
$ curl -s https://envshare.dev/api/v1/secret/HdPbXgpvUvNk43oxSdK97u
```

```json
{
  "data": {
    "secret": "Hello",
    "remainingReads": 1
  }
}
```
