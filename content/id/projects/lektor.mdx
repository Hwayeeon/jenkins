## Tampilan Bisnis

### Ringkasan Proyek
Lektor adalah platform web yang dirancang untuk memodernisasi penggalangan dana komunitas dengan mengubah alur kerja pemesanan manual menjadi pengalaman online yang lancar. Platform ini memungkinkan komunitas gereja dan penjual makanan skala kecil untuk menjual produk secara digital sambil melacak penjualan secara otomatis, sehingga penggalangan dana menjadi lebih cepat, akurat, dan menguntungkan.

### Tantangan
Sebelum Lektor:

- Pesanan dikelola secara manual melalui telepon atau tatap muka.
- Pelacakan pembayaran dan pesanan rentan kesalahan dan lambat.
- Jangkauan dan visibilitas pelanggan terbatas.

Masalah utama: bagaimana menciptakan sistem pemesanan online yang skalabel dan terpercaya tanpa infrastruktur yang rumit.

### Solusi
Saya membangun platform web ringan dan intuitif di mana:

- Penjual mengunggah daftar produk beserta gambar dan harga.
- Pelanggan dapat menjelajahi barang, menambahkannya ke keranjang belanja, dan melakukan checkout tanpa perlu membuat akun.
- Pesanan secara otomatis dikirim ke Google Sheets untuk pelacakan real-time:
  - Nama dan nomor telepon pelanggan
  - Barang yang dipilih dan jumlahnya
  - Total penjualan
  - Status pembayaran

Pengaturan ini menghilangkan entri manual sambil menjaga semua data terpusat dan dapat diekspor.

### Dampak & Hasil

- Pendapatan meningkat lebih dari 30% untuk komunitas yang menggunakan Lektor.
- Penggalangan dana menjadi lebih efisien dengan pelacakan otomatis dan pengurangan kesalahan.
- Baik pihak yang bertanggung jawab (PIC) maupun pembeli menemukan sistem ini mudah dan nyaman.

> Ringkasan: Lektor menunjukkan bagaimana aplikasi web kecil namun dirancang dengan baik dapat mengubah penggalangan dana offline menjadi sistem yang skalabel, otomatis, dan menguntungkan, memudahkan hidup bagi penjual dan pembeli.

## Tampilan Developer

### Ringkasan Teknis
Lektor adalah aplikasi web full-stack yang dibangun dengan teknologi modern, menunjukkan praktik pengembangan web yang skalabel, dapat dipelihara, dan berperforma tinggi.

### Arsitektur & Stack Teknologi
- Frontend: Next.js (React) untuk rendering sisi server dan antarmuka responsif
- Gaya: Tailwind CSS untuk desain berorientasi mobile
- Backend: Rute API untuk validasi dan pemrosesan pesanan
- Penyimpanan Data: Google Sheets API v4 untuk pelacakan pesanan secara real-time
- Deployment: Vercel untuk CI/CD dan jaringan tepi global

### Tantangan Teknis Utama

#### 1. Integrasi Google Sheets
- Menerapkan alur otentikasi OAuth 2.0 dan Sheets API v4
- Memungkinkan sinkronisasi pesanan real-time tanpa database tradisional
- Menambahkan validasi data otomatis, pemformatan, dan penanganan kesalahan

#### 2. Sistem Manajemen Pesanan
- Validasi formulir sisi klien dengan skema Zod
- Sanitasi dan verifikasi data sisi server
- Pembuatan ID pesanan unik dan cap waktu
- Memastikan perilaku transaksi untuk konsistensi

#### 3. Optimasi Kinerja
- Optimasi gambar dengan komponen Image Next.js
- Pemuatan lambat katalog produk
- Pencarian dan penyaringan yang di-debounce
- Caching sisi klien untuk pengalaman pengguna yang lebih cepat

#### 4. Desain Responsif
- Tata letak mobile-first dan kontrol ramah sentuhan
- Titik pemecahan untuk berbagai ukuran layar
- Peningkatan progresif untuk browser lama

#### 5. Kualitas Kode & Praktik Terbaik
- TypeScript untuk keamanan tipe
- Arsitektur berbasis komponen dengan komponen UI yang dapat digunakan ulang
- Hook kustom untuk logika bersama
- Variabel lingkungan untuk konfigurasi
- Batas kesalahan untuk penanganan kegagalan yang elegan

### Pertimbangan Skalabilitas
- Lapisan data modular untuk migrasi mudah ke database tradisional
- Desain berbasis API memungkinkan integrasi aplikasi seluler di masa depan
- Desain server stateless untuk skalabilitas horizontal

### Alur Pengguna
- Pengguna mengunjungi situs web
- Tidak memerlukan login
- Pilih item → Tambahkan ke keranjang → Checkout
- Pesan otomatis ke PIC
- Pesanan tercatat di Google Sheets → Transaksi selesai

| Column            | Description                |
| ----------------- | -------------------------- |
| nomor             | Serial number              |
| id_pembelian      | Unique order ID            |
| nama pembeli      | Customer name              |
| nomor telepon     | Phone number               |
| order item full   | Full list of ordered items |
| total order item  | Total amount               |
| status pembayaran | Payment status             |
