## Tampilan Bisnis

### Ikhtisar Proyek
Lektor adalah platform web yang saya bangun untuk mendigitalisasi penggalangan dana komunitas dengan mengubah alur pemesanan manual menjadi ekosistem pemesanan online yang mulus dan otomatis. Proyek ini dirancang untuk membantu komunitas gereja dan penjual makanan skala kecil menghasilkan pendapatan secara lebih efisien dengan menghubungkan penjual langsung dengan pengunjung lokal yang dapat memesan secara online.

### Tantangan
Sebelum Lektor, pesanan penggalangan dana dilakukan secara manual — melalui telepon atau tatap muka — yang menyebabkan:

- Memakan waktu dan rentan terjadi kesalahan
- Sulit untuk melacak atau mengelola pesanan
- Limitasi jangkauan pelanggan dan visibilitas

Masalah utama adalah _bagaimana mengubah alur offline yang tidak terorganisir menjadi pengalaman online yang terpercaya dan skalabel tanpa infrastruktur yang rumit_.

### Solusi Saya
Saya merancang dan membangun platform pemesanan web ringan di mana:
- Penjual mengunggah _daftar produk_ dengan harga dan gambar
- Pengunjung _dapat menelusuri dan memesan secara instan_ dalam alur yang mudah digunakan
- Pesanan otomatis sinkron ke Google Sheets, mengumpulkan:
  - Nama pelanggan
  - Nomor telepon
  - Item yang dipilih
  - Total penjualan

Pengaturan ini menghapus pekerjaan masukan manual sementara mempertahankan data yang terpusat dan dapat diekspor.

### Dampak & Hasil
Proyek ini bukan hanya peningkatan fungsi — ia menciptakan nilai bisnis yang dapat diukur:

- **Kinerja penjualan meningkat signifikan**
- Komunitas melihat **peningkatan pendapatan hingga 30%+** setelah menerapkan Lektor
- Efisiensi proses meningkat dengan pelacakan otomatis dan pengurangan kesalahan

Lektor membuktikan bagaimana otomatisasi web yang berpikir matang dapat mengubah upaya penggalangan dana lokal menjadi saluran pendapatan yang skalabel.

## Tampilan Developer

### Ikhtisar Teknis
Lektor adalah aplikasi web full-stack yang mendemonstrasikan praktik pengembangan web modern, mengintegrasikan berbagai teknologi untuk menciptakan solusi e-commerce lengkap untuk penggalangan dana.

### Arsitektur & Tech Stack
- **Frontend**: Next.js dengan React untuk server-side rendering dan performa optimal
- **Styling**: Tailwind CSS untuk desain responsif mobile-first
- **Backend**: API routes untuk pemrosesan dan validasi pesanan
- **Database/Storage**: Integrasi Google Sheets API untuk sinkronisasi data real-time
- **Deployment**: Vercel untuk CI/CD otomatis dan distribusi edge network

### Tantangan Teknis Utama

#### 1. Integrasi Google Sheets
Mengimplementasikan OAuth 2.0 authentication flow dan Google Sheets API v4 untuk:
- Sinkronisasi pesanan real-time tanpa infrastruktur database
- Validasi dan formatting data otomatis
- Error handling untuk API rate limits dan network failures

#### 2. Sistem Manajemen Pesanan
Membangun pipeline pemrosesan pesanan yang robust:
- Validasi form client-side dengan Zod schemas
- Sanitasi dan verifikasi data server-side
- Behavior mirip transaction untuk memastikan konsistensi data
- Generasi timestamp dan order ID otomatis

#### 3. Optimasi Performa
- Optimasi gambar menggunakan Next.js Image component
- Lazy loading untuk katalog produk
- Debounced search dan filtering
- Client-side caching untuk UX yang lebih baik

#### 4. Desain Responsif
Mencapai interface mobile-first yang fully responsive:
- Sistem breakpoint untuk berbagai ukuran layar
- Kontrol yang dioptimasi untuk perangkat sentuh
- Strategi progressive enhancement

### Kualitas Kode & Best Practices
- TypeScript untuk type safety di seluruh codebase
- Arsitektur berbasis komponen untuk reusability
- Custom hooks untuk logic yang dibagikan
- Manajemen environment variable untuk konfigurasi
- Error boundaries untuk penanganan failure yang graceful

### Pertimbangan Skalabilitas
Meskipun Google Sheets berfungsi sebagai data layer untuk MVP, arsitektur dirancang dengan modularitas:
- Abstracted data access layer untuk migrasi mudah ke database tradisional
- Desain API-first memungkinkan integrasi aplikasi mobile di masa depan
- Desain server stateless untuk horizontal scaling

Proyek ini menunjukkan tidak hanya kemampuan untuk mengirimkan fitur, tetapi untuk merancang solusi yang maintainable dan scalable sambil menyeimbangkan kendala teknis dengan kebutuhan bisnis.
